# TRD (Technical Requirements Document) — 경량화 버전

## ReMedi — LLM 기반 복약·건강관리 서비스

| 항목 | 내용 |
| --- | --- |
| 관련 PRD | `PRD_ReMedi.md` |
| 문서 버전 | v1.0 (2026-07-06) |

> 데이터 모델, API 명세, 상세 아키텍처는 별도 문서에서 관리합니다.
> 이 문서는 각 기능 단위(T-그룹)별 **입력 항목 / 출력·노출 항목 / 성공요건(Pass Criteria)**만 정의합니다.
> 흐름·구현 방식(어떤 라이브러리로, 어떤 순서로 처리할지)은 개발자 재량에 맡깁니다.
> 각 T-그룹은 PRD의 F-그룹과 번호가 1:1 대응됩니다.

## 1. 회원/인증

### T-AUTH-1. 이메일 회원가입
- **입력**: 이메일, 비밀번호, 닉네임, 이름, 휴대폰번호, 생년월일, 성별
- **출력/노출**: 인증번호 발송 상태, 닉네임 중복확인 결과, 가입 완료 시 Access/Refresh 토큰
- **성공요건**
  - 이메일·휴대폰 인증 미완료 시 가입이 완료되지 않는다
  - 닉네임 중복 시 가입이 거부된다
  - 저장소를 조회해도 비밀번호 평문이 노출되지 않는다
  - 가입 완료 즉시 별도 로그인 없이 홈 진입이 가능하다
- *(관련 PRD: F-AUTH-1 / 관련 REQ: REQ-USER-001~004, REQ-UI-USER-001~002)*

### T-AUTH-2. Google 소셜 로그인
- **입력**: Google ID Token
- **출력/노출**: 로그인/가입 처리 결과, 추가 입력 요청 여부, Access/Refresh 토큰
- **성공요건**
  - 위조·만료된 ID Token은 거부된다(인증 실패 처리)
  - 기존 가입자는 즉시 로그인, 신규 사용자는 즉시 프로필 생성 및 로그인 완료
  - 닉네임 중복은 사용자 개입 없이 자동 해결된다
- *(관련 PRD: F-AUTH-2 / 관련 REQ: REQ-USER-022~028, REQ-AUTH-005, REQ-UI-USER-003~005)*

### T-AUTH-3. 로그인 유지 및 토큰 갱신
- **입력**: 로그인 자격증명 또는 Refresh Token
- **출력/노출**: Access/Refresh Token, 만료 시각
- **성공요건**
  - Access Token 만료 후에도 재로그인 없이 서비스 이용이 끊기지 않는다
  - Refresh Token까지 무효화된 경우에만 로그인 화면으로 이동한다
  - 재발급 시 이전 Refresh Token은 재사용 불가능하다
- *(관련 PRD: F-AUTH-3 / 관련 REQ: REQ-AUTH-001~004)*

### T-AUTH-4. 로그아웃
- **입력**: 현재 기기의 푸시 구독 식별자
- **출력/노출**: 로그아웃 처리 결과
- **성공요건**
  - 로그아웃한 기기로는 이후 개인화 알림이 발송되지 않는다
  - 재접속 시 이전 사용자의 데이터가 화면에 남아있지 않다
  - 서버 처리 실패 시에도 클라이언트 로그아웃은 완료된다
- *(관련 PRD: F-AUTH-4 / 관련 REQ: REQ-USER-019~021)*

### T-AUTH-5. 가족연결(계정 간 연결)
- **입력**: 초대 토큰 발급 요청, 초대 토큰/이메일 수락, 관계동의 여부, 순응도공유동의 여부
- **출력/노출**: 연결된 서포트 그룹 목록, 상대방의 복약 순응도 %(세부 복약정보 미포함)
- **성공요건**
  - 관계 해제 시 즉시 양쪽 목록에서 사라진다
  - 동일 대상과 중복 연결되지 않는다
  - 초대 토큰은 추측 불가능한 값으로 발급되며, 만료시간 경과 또는 1회 사용 후 재사용이 불가능하다
  - 응답 스키마에 약품명·복용시각·사유 등 세부 복약정보가 포함되지 않는다
  - 관계 동의와 순응도 공유 동의는 각각 독립적으로 on/off 가능하다
  - 비회원 초대 시, 가입 완료 즉시 별도 재입력 없이 연결이 자동 성립된다
- *(관련 PRD: F-AUTH-5 / 관련 REQ: REQ-LINK-001)*

### T-AUTH-6. 가족구성원관리
- **입력**: 관리 대상 프로필 생성 요청(이름 등 기본정보)
- **출력/노출**: 계정 내 프로필 목록, 현재 활성 프로필 상태
- **성공요건**
  - 상대방 계정 생성이나 수락 절차 없이 프로필이 즉시 추가된다
  - 프로필별 복약·건강 데이터가 서로 분리되어 저장·조회된다(`profile_id` 기준 스키마)
  - 프로필 추가 시 기존 테이블 구조 변경(마이그레이션) 없이 반영 가능하다
- *(관련 PRD: F-AUTH-6 / 관련 REQ: 신규)*

## 2. 복약·건강기록 관리

### 2-1. 복약·문서 인식

#### T-MED-1. 알약 인식 및 복약 스케줄 등록
- **입력**: 알약 사진 또는 처방전 PDF/이미지
- **출력/노출**: 식별 후보 리스트(약품명, 매칭률), 최종 선택된 약품 상세정보, 등록된 복약 스케줄
- **성공요건**
  - 신뢰도가 낮거나 후보가 여러 개면 사용자 최종 선택 없이는 등록되지 않는다
  - 식별 실패 시 수동 검색으로 전환되며 등록 자체가 막히지 않는다
  - 스케줄 등록은 시간 선택 외 추가 입력 없이 완료 가능해야 한다
- *(관련 PRD: F-MED-1 / 관련 REQ: REQ-MED-001~004, REQ-MED-006~010)*

#### T-MED-2. 약품 안전 정보 안내
- **입력**: 신규 등록 약품, 기존 복약·과거력 데이터
- **출력/노출**: 상충 위험 경고 카드
- **성공요건**
  - 상충 위험이 있는 경우 등록 완료 화면에서 별도 진입 없이 즉시 노출된다
- *(관련 PRD: F-MED-2 / 관련 REQ: REQ-MED-005)*

#### T-DOC-1. 처방전·진료기록·복약안내문 인식 및 등록
- **입력**: 문서 사진 또는 PDF
- **출력/노출**: 추출된 약품/투약정보/진단명 목록(수정 가능한 폼), 확정 후 2/3단계 프로필 반영 결과
- **성공요건**
  - 사용자 컨펌 없이는 DB에 최종 저장되지 않는다
  - 확정 시 별도 재입력 없이 관련 단계(복약정보/과거력)에 자동 반영된다
- *(관련 PRD: F-DOC-1 / 관련 REQ: REQ-DOC-001~004, REQ-DOC-006~008)*

#### T-DOC-2. 인식 데이터 기반 맞춤 가이드
- **입력**: 확정된 진료기록/복약안내문 데이터
- **출력/노출**: 약물·음식 시너지/부작용 안내 카드
- **성공요건**
  - 문서 등록 직후 별도 요청 없이 관련 안내가 노출된다
- *(관련 PRD: F-DOC-2 / 관련 REQ: REQ-DOC-009)*

#### T-DOC-3. 원본 문서 보관 및 폐기
- **입력**: 삭제 요청(사용자)
- **출력/노출**: 날짜별 원본 갤러리, 삭제 완료 결과
- **성공요건**
  - 삭제 요청 시 원본과 추출 데이터가 복구 불가능하게 제거된다
- *(관련 PRD: F-DOC-3 / 관련 REQ: REQ-DOC-005, REQ-DOC-010)*

### 2-2. 목표·순응도 관리

#### T-GOAL-1. 목표 및 관심 질병 설정
- **입력**: 목표 카테고리, 목표 수치, 기간, 관심/보유 질병
- **출력/노출**: 등록된 목표·질병 목록(CRUD)
- **성공요건**
  - 목표는 등록·수정·삭제가 모두 가능해야 한다
  - 질병은 검색 또는 칩 선택 중 하나로 1회 조작 내 등록 가능해야 한다
- *(관련 PRD: F-GOAL-1 / 관련 REQ: REQ-GOAL-001, REQ-GUIDE-001)*

#### T-GOAL-2. 목표 기반 맞춤 가이드 생성·갱신
- **입력**: 목표 수치, 신체정보, 지병 정보
- **출력/노출**: 식단/운동 가이드, 재생성 진행 상태 표시
- **성공요건**
  - 지병이 있는 경우 해당 지병에 위험한 항목이 추천 결과에 포함되지 않는다
  - 목표 변경 시 가이드가 자동 재생성되고, 재생성 중임이 화면에 표시된다
- *(관련 PRD: F-GOAL-2 / 관련 REQ: REQ-GOAL-002~004)*

#### T-GOAL-3. 목표 달성 리포트
- **입력**: 없음(스케줄 기반 자동 생성)
- **출력/노출**: 주간/월간 리포트 카드(달성률, 코멘트)
- **성공요건**
  - 정해진 주기(주간/월간)에 누락 없이 리포트가 생성된다
  - 달성률 저조 항목에는 코멘트가 함께 노출된다
- *(관련 PRD: F-GOAL-3 / 관련 REQ: REQ-GOAL-005)*

#### T-ADH-1. 복약 순응도 집계 및 시각화
- **입력**: 복용 완료 체크 기록
- **출력/노출**: 당일 위젯(진행률), 히트맵, 순응도 %
- **성공요건**
  - 지연 복용은 미복용이 아닌 부분 인정으로 반영된다
  - 순응도 수치는 실시간(당일) 및 배치(주/월) 두 기준 모두 확인 가능하다
- *(관련 PRD: F-ADH-1 / 관련 REQ: REQ-ADH-001, REQ-CALC-001~004)*

#### T-ADH-2. 순응도 기반 동기부여 피드백
- **입력**: 순응도 수치, 3단계 정보
- **출력/노출**: 피드백 메시지, 시너지/주의 음식 카드, 가족 알림
- **성공요건**
  - 고순응도/저순응도에 따라 메시지 톤이 달라진다
  - 순응도가 기준치 이하로 지속되면 가족에게 알림이 전달된다
- *(관련 PRD: F-ADH-2 / 관련 REQ: REQ-ADH-002~004)*

### 2-3. 맞춤 건강정보/가이드 제공

#### T-GUIDE-1. 질병별 통합 정보 대시보드
- **입력**: 등록된 질병, 복약 정보
- **출력/노출**: 질병별 카드(복약정보+가이드+주의사항+면책조항)
- **성공요건**
  - 상호작용/금기사항은 시각적으로 구분되어 노출된다
  - 모든 카드에 면책조항이 함께 노출된다
- *(관련 PRD: F-GUIDE-1 / 관련 REQ: REQ-GUIDE-002~005)*

#### T-INFO-1. 약품 상세정보 조회
- **입력**: 검색어 또는 등록된 복약 목록 선택
- **출력/노출**: 약품 상세정보, (3단계 미등록 시) 등록 유도 카드
- **성공요건**
  - 3단계 등록 여부에 따라 노출되는 정보의 구체성이 달라진다
- *(관련 PRD: F-INFO-1 / 관련 REQ: REQ-INFO-005~007)*

#### T-INFO-2. 약품 안전성 챗봇 질의응답
- **입력**: 자연어 질문
- **출력/노출**: 사용자 맥락이 반영된 답변
- **성공요건**
  - 사용자가 자신의 지병/복약 정보를 다시 설명하지 않아도 답변에 반영된다
- *(관련 PRD: F-INFO-2 / 관련 REQ: REQ-INFO-008)*

## 3. LLM/AI 서비스

### T-LLM-1. 안전 프롬프트 및 면책조항 공통 정책
- **입력**: 없음(모든 LLM 응답에 공통 적용)
- **출력/노출**: 면책조항 컴포넌트, 안전 프롬프트가 반영된 LLM 응답
- **성공요건**
  - 면책조항은 사용자가 끄거나 숨길 수 없다
  - 신규로 추가되는 LLM 기능도 별도 조치 없이 이 정책이 기본 적용된다
- *(관련 PRD: F-LLM-1 / 관련 REQ: REQ-LLM-001~002)*

### T-LLM-2. AI 챗봇 상담
- **입력**: 자연어 질문(텍스트/음성)
- **출력/노출**: 스트리밍 답변, 응급 시 Fallback 메시지, 음성 출력
- **성공요건**
  - 응급 암시 키워드 입력 시 일반 답변 대신 즉시 Fallback 메시지가 노출된다
  - 이전 대화 맥락이 같은 세션 내에서 유지된다
- *(관련 PRD: F-LLM-2 / 관련 REQ: REQ-BOT-001~005)*

### T-LLM-3. 건강 콘텐츠 생성 파이프라인
- **입력**: 질환 태그, 생활 패턴 데이터
- **출력/노출**: 규격화된 팁 카드(JSON 기반)
- **성공요건**
  - 동일 질환 콘텐츠가 당일 이미 존재하면 재생성 없이 캐시가 재사용된다
  - 다빈도 질환은 사용자 접속 이전에 미리 생성되어 있어야 한다
- *(관련 PRD: F-LLM-3 / 관련 REQ: REQ-CONT-001~004)*

### T-LLM-4. 공유용 카드뉴스 이미지 생성
- **입력**: 규격화된 콘텐츠(제목/요약 등)
- **출력/노출**: 공유 가능한 이미지 파일
- **성공요건**
  - 생성된 이미지에도 축약된 면책 문구가 포함된다
  - 공유 버튼 클릭 후 별도 대기 없이 다운로드/공유 시트가 열린다
- *(관련 PRD: F-LLM-4 / 관련 REQ: REQ-CONT-005)*

### T-LLM-5. 관리자 안전 프롬프트 관리
- **입력**: 신규 프롬프트 문구(관리자)
- **출력/노출**: 현재 활성 프롬프트, 버전 이력
- **성공요건**
  - 변경 시 기존 버전을 덮어쓰지 않고 이력이 남는다
  - 활성화 즉시 신규 요청부터 반영된다(배포 불필요)
- *(관련 PRD: F-LLM-5 / 관련 REQ: REQ-ADM-001)*

### T-LLM-6. 건강 뉴스 수집 및 개인화 매칭
